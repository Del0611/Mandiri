# 🌐 Materi Praktikum: Konfigurasi Dasar Router

## 🎯 Tujuan Praktikum

Setelah menyelesaikan praktikum ini, peserta didik mampu:

- 📌 Memahami fungsi router dalam jaringan komputer.
- 📌 Melakukan konfigurasi IP Address pada router.
- 📌 Mengaktifkan layanan DHCP Server.
- 📌 Menghubungkan perangkat klien ke router.
- 📌 Menguji konektivitas jaringan menggunakan perintah *ping*.

---

📌 Memahami Fungsi Router dalam Jaringan Komputer

Router adalah perangkat jaringan yang berfungsi untuk menghubungkan dua atau lebih jaringan yang berbeda serta mengatur lalu lintas data agar dapat sampai ke tujuan yang benar. Router bekerja dengan membaca alamat IP tujuan dan menentukan jalur terbaik untuk mengirimkan paket data.

Fungsi utama router:

🌐 Menghubungkan jaringan lokal (LAN) dengan internet.
🔀 Mengarahkan paket data antar jaringan.
📡 Membagi koneksi internet ke banyak perangkat.
🔒 Mengatur keamanan jaringan melalui firewall dan fitur keamanan lainnya.
📋 Mengelola lalu lintas data agar jaringan lebih efisien.
📌 Melakukan Konfigurasi IP Address pada Router

Konfigurasi IP Address pada router adalah proses pemberian alamat IP pada interface router agar dapat berkomunikasi dengan perangkat lain dalam jaringan.

Tujuan konfigurasi IP Address:

🌍 Memberikan identitas pada router di dalam jaringan.
🔗 Memungkinkan router berkomunikasi dengan perangkat klien.
📡 Menjadi gateway atau pintu keluar masuk data dalam jaringan.

Contoh konfigurasi:

Parameter	Nilai
IP Address	192.168.10.1
Subnet Mask	255.255.255.0

Pada contoh tersebut, router akan menjadi gateway bagi perangkat yang berada pada jaringan 192.168.10.0/24.

📌 Mengaktifkan Layanan DHCP Server

DHCP (Dynamic Host Configuration Protocol) adalah layanan yang memberikan alamat IP secara otomatis kepada perangkat yang terhubung ke jaringan.

Manfaat DHCP Server:

⚡ Mempercepat proses konfigurasi jaringan.
📝 Mengurangi kesalahan pengaturan IP secara manual.
👥 Memudahkan pengelolaan banyak perangkat dalam jaringan.
🔄 Membagikan IP Address secara otomatis saat perangkat terhubung.

Contoh alokasi DHCP:

Parameter	Nilai
Rentang IP	192.168.10.10 – 192.168.10.100
Gateway	192.168.10.1
DNS Server	8.8.8.8

Dengan DHCP aktif, pengguna tidak perlu mengatur IP Address secara manual pada setiap perangkat.

📌 Menghubungkan Perangkat Klien ke Router

Perangkat klien adalah perangkat yang menggunakan layanan jaringan yang disediakan oleh router, seperti komputer, laptop, smartphone, atau printer jaringan.

Cara menghubungkan klien ke router:

🔌 Menggunakan kabel LAN melalui port Ethernet.
📶 Menggunakan jaringan nirkabel (Wi-Fi).
⚙️ Mengatur jaringan klien agar memperoleh IP secara otomatis (DHCP) atau manual.

Tujuan menghubungkan klien ke router:

🌐 Mengakses internet.
📂 Berbagi data dalam jaringan lokal.
🖨️ Mengakses perangkat bersama seperti printer dan server.
📌 Menguji Konektivitas Jaringan Menggunakan Perintah Ping

Ping adalah perintah yang digunakan untuk menguji apakah suatu perangkat dapat berkomunikasi dengan perangkat lain melalui jaringan.

Perintah ini mengirimkan paket data kecil (ICMP Echo Request) ke alamat tujuan dan menunggu balasan (ICMP Echo Reply).

Contoh perintah:

ping 192.168.10.1

Tujuan pengujian ping:

📡 Memastikan perangkat terhubung ke jaringan.
🔍 Memeriksa apakah router dapat diakses.
⚡ Mengukur waktu respons jaringan (latency).
🛠️ Membantu proses troubleshooting jaringan.

Contoh hasil berhasil:

Reply from 192.168.10.1: bytes=32 time<1ms TTL=64
Reply from 192.168.10.1: bytes=32 time<1ms TTL=64

Contoh hasil gagal:

Request timed out.

Jika muncul pesan Request timed out, kemungkinan terdapat masalah pada kabel jaringan, konfigurasi IP, atau perangkat jaringan yang digunakan.

# 🔧 D. Langkah Kerja Praktikum

## 🔹 Langkah 1: Persiapan Perangkat

1. 🔌 Hubungkan adaptor daya ke router.
2. 💡 Pastikan lampu indikator Power menyala normal.
3. 🌐 Hubungkan laptop ke port LAN router menggunakan kabel UTP.
4. 🖥️ Nyalakan laptop dan router.

---

## 🔹 Langkah 2: Mengakses Router

1. 🌍 Buka aplikasi browser atau WinBox (untuk MikroTik).
2. ✏️ Masukkan alamat IP default router.

Contoh:

```text
192.168.88.1
```

3. 🔑 Login menggunakan akun administrator.

Contoh:

```text
Username : admin
Password : (kosong)
```

---

## 🔹 Langkah 3: Konfigurasi IP Address Router

1. ⚙️ Masuk ke menu **IP → Addresses**.
2. ➕ Klik tombol **Add (+)**.
3. Masukkan konfigurasi berikut:

| Parameter | Nilai |
|-----------|---------|
| IP Address | 192.168.10.1/24 |
| Interface | ether2 |

4. 💾 Klik **Apply** dan **OK**.

---

## 🔹 Langkah 4: Konfigurasi DHCP Server

1. 📡 Buka menu **IP → DHCP Server**.
2. Klik **DHCP Setup**.
3. Pilih interface yang digunakan untuk jaringan lokal.
4. Tentukan rentang alamat IP.

Contoh:

| Parameter | Nilai |
|-----------|---------|
| Network | 192.168.10.0/24 |
| Gateway | 192.168.10.1 |
| IP Pool | 192.168.10.10 - 192.168.10.100 |
| DNS Server | 8.8.8.8 |

5. 💾 Selesaikan konfigurasi hingga DHCP Server aktif.

---

## 🔹 Langkah 5: Verifikasi Konfigurasi

1. ⌨️ Buka Command Prompt pada komputer klien.
2. Jalankan perintah:

```bash
ipconfig
```

3. ✅ Pastikan komputer memperoleh IP Address secara otomatis dari router.

Contoh:

```text
IPv4 Address : 192.168.10.10
Default Gateway : 192.168.10.1
```

---

## 🔹 Langkah 6: Pengujian Konektivitas

1. ⌨️ Jalankan perintah berikut:

```bash
ping 192.168.10.1
```

2. Perhatikan hasil pengujian.

Contoh hasil berhasil:

```text
Reply from 192.168.10.1: bytes=32 time<1ms TTL=64
Reply from 192.168.10.1: bytes=32 time<1ms TTL=64
Reply from 192.168.10.1: bytes=32 time<1ms TTL=64
Reply from 192.168.10.1: bytes=32 time<1ms TTL=64
```

---

## 🔹 Langkah 7: Analisis Hasil

### ✅ Jika Berhasil

- Router dapat diakses dari komputer klien.
- DHCP Server berjalan dengan baik.
- Komputer mendapatkan IP Address secara otomatis.
- Koneksi jaringan berfungsi normal.

### ❌ Jika Gagal

Periksa kembali:

- 🔌 Kabel jaringan terpasang dengan benar.
- 🌐 Konfigurasi IP Address router sesuai.
- 📡 DHCP Server aktif.
- ⚙️ Interface jaringan aktif.
- 🔥 Firewall tidak memblokir koneksi.
