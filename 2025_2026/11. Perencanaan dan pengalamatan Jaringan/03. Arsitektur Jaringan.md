# 🌐 Materi 3: Arsitektur Jaringan – Client-Server dan Peer-to-Peer

Setelah memahami topologi atau bentuk fisik jaringan pada Materi 2, kita sekarang akan mempelajari **Arsitektur Jaringan (Logis)**. Arsitektur jaringan menentukan bagaimana peran dan kedudukan masing-masing perangkat dalam bertukar data dan mengelola sumber daya.

Dalam dunia jaringan, arsitektur secara umum dibagi menjadi dua model utama: **Client-Server** dan **Peer-to-Peer (P2P)**.

---

## 📌 1. Memahami Konsep Arsitektur Client-Server

### Definisi

**Client-Server** adalah model arsitektur jaringan di mana fungsi perangkat dibagi secara jelas menjadi dua peran:

1. **Server (Penyedia):** Komputer berspesifikasi tinggi yang bertindak sebagai pusat data dan bertugas menyediakan layanan, validasi keamanan, serta memproses permintaan.
2. **Client (Pengguna):** Perangkat akhir (*end-device* seperti PC atau Laptop) yang digunakan oleh pengguna untuk meminta data atau layanan dari server.


<img width="3000" height="2000" alt="Client server" src="https://github.com/user-attachments/assets/3ac6d981-0d57-4008-a27e-4ef8e3888484" />


### Karakteristik Jaringan Client-Server:

* **Sentralisasi:** Seluruh data, hak akses, dan manajemen jaringan dipusatkan di satu atau beberapa server utama.
* **Keamanan Tinggi:** Pengawasan akses data lebih mudah dikontrol karena otentikasi pengguna dilakukan secara terpusat oleh server.
* **Skalabilitas Baik:** Sangat cocok untuk jaringan berskala besar (perusahaan, sekolah, atau instansi pemerintah) karena pengelolaan ribuan komputer client dapat dikontrol dari satu titik.
* **Ketergantungan Tinggi:** Jika server pusat mengalami kerusakan (*down*), maka seluruh PC client tidak dapat mengakses layanan atau data tersebut.

---

## 📌 2. Memahami Konsep Arsitektur Peer-to-Peer (P2P)

### Definisi

**Peer-to-Peer (P2P)** adalah model arsitektur jaringan di mana setiap komputer yang terhubung memiliki kedudukan, hak, dan kewajiban yang sama (*equal*). Tidak ada komputer yang didedikasikan khusus sebagai server tunggal.


<img width="3000" height="2000" alt="peer to peer" src="https://github.com/user-attachments/assets/fd76e735-8354-43b8-a54c-c4027940a3f7" />


### Karakteristik Jaringan Peer-to-Peer:

* **Desentralisasi:** Data tersebar di masing-masing komputer pengguna. Komputer A bisa meminta data ke Komputer B (bertindak sebagai client), dan di saat yang sama Komputer A juga bisa membagikan filenya ke Komputer C (bertindak sebagai server).
* **Biaya Rendah:** Tidak membutuhkan komputer server khusus dengan spesifikasi mahal maupun sistem operasi server (seperti *Windows Server* atau *Linux Enterprise*).
* **Implementasi Mudah:** Sangat cocok untuk jaringan skala kecil (kurang dari 10 perangkat), seperti transfer file antar-laptop di ruangan yang sama.
* **Keamanan Tersebar:** Keamanan jaringan bergantung pada masing-masing pengguna komputer, sehingga risiko kebocoran data atau infeksi virus lebih tinggi jika salah satu pengguna lalai.

---

### 📊 Tabel Perbandingan: Client-Server vs Peer-to-Peer

| Parameter | Client-Server | Peer-to-Peer (P2P) |
| --- | --- | --- |
| **Pusat Kendali** | Terpusat (*Centralized*) di Server | Tersebar (*Decentralized*) di setiap PC |
| **Biaya Instalasi** | Mahal (Butuh perangkat & OS Server) | Murah (Cukup PC biasa & kabel) |
| **Kapasitas Pengguna** | Sangat Besar (Skala Menengah - Luas) | Terbatas (Skala Kecil, < 10 PC) |
| **Keamanan** | Sangat Terjamin & Terkontrol | Kurang Terjamin (Tergantung user) |
| **Penyimpanan Data** | Terpusat di Harddisk Server | Tersebar di masing-masing Harddisk PC |

## 🏗️ 3. Membangun Jaringan Peer-to-Peer Sederhana di Cisco Packet Tracer

Dalam praktik ini, kita akan menghubungkan dua buah PC secara langsung tanpa menggunakan Switch.

### Langkah-Langkah:

1. Buka aplikasi **Cisco Packet Tracer**.
2. Tambahkan **2 unit PC** ke area kerja (PC0 dan PC1).
3. Pilih menu **Connections** (ikon petir), lalu pilih kabel **Copper Cross-Over** (ikon garis putus-putus hitam).
> ⚠️ **Aturan Pengabelan:** Kabel *Cross-Over* digunakan untuk menghubungkan dua perangkat yang setingkat/identik (PC ke PC, Router ke Router, Switch ke Switch).


4. Hubungkan port `FastEthernet0` pada PC0 ke port `FastEthernet0` pada PC1.
5. Lakukan konfigurasi IP Address pada kedua PC:
* Klik **PC0** > tab **Desktop** > **IP Configuration** > Isi IP: `192.168.1.1` dengan Subnet Mask: `255.255.255.0`.
* Klik **PC1** > tab **Desktop** > **IP Configuration** > Isi IP: `192.168.1.2` dengan Subnet Mask: `255.255.255.0`.

## 🏗️ 4. Membangun Jaringan Client-Server di Cisco Packet Tracer

Pada praktik ini, kita akan mensimulasikan sebuah jaringan lokal sekolah yang memiliki satu Server Pusat (Penyedia Layanan Web/HTTP) dan dua PC Client yang dihubungkan melalui sebuah Switch.

### Langkah-Langkah:

1. Tambahkan perangkat berikut ke area kerja:
* **1 unit Server** (Pilih di menu *End Devices* > *Server*)
* **2 unit PC** (PC0 dan PC1)
* **1 unit Switch** (Pilih tipe *2960*)


2. Hubungkan semua perangkat menggunakan kabel **Copper Straight-Through** (ikon garis hitam solid):
* Server ke Switch (port `FastEthernet0/1`)
* PC0 ke Switch (port `FastEthernet0/2`)
* PC1 ke Switch (port `FastEthernet0/3`)


3. Lakukan konfigurasi IP Address secara statis:
* **Server:** IP `192.168.2.1` | Subnet Mask `255.255.255.0`
* **PC0:** IP `192.168.2.2` | Subnet Mask `255.255.255.0`
* **PC1:** IP `192.168.2.3` | Subnet Mask `255.255.255.0`


4. **Mengaktifkan Layanan Web (HTTP) di Server:**
* Klik pada **Server**, masuk ke tab **Services**.
* Pilih menu **HTTP** di panel sebelah kiri.
* Pastikan opsi HTTP dan HTTPS berada pada posisi **On**.
* (Opsional) Anda dapat mengedit file `index.html` untuk mengubah teks sambutan website.

## 🧪 5. Menguji Komunikasi Antar-Perangkat dan Layanan Jaringan

Setelah kedua arsitektur selesai dibangun, kita harus menguji aspek fungsionalitasnya.

### A. Pengujian Konektivitas Dasar (Ping Test)

1. Klik **PC0** pada jaringan Client-Server.
2. Buka **Command Prompt** di tab **Desktop**.
3. Lakukan ping ke alamat IP Server dengan mengetik:
```bash
ping 192.168.2.1

```


4. Jika mendapati respon `Reply from 192.168.2.1...`, berarti PC Client telah berhasil terhubung secara fisik dan logis dengan Server.

### B. Pengujian Fungsi Layanan (Web Browser Test)

Untuk membuktikan bahwa arsitektur *Client-Server* berjalan seutuhnya, kita akan mengakses layanan web yang disediakan oleh Server melalui PC Client.

1. Klik kembali **PC0**, lalu pilih menu **Web Browser** di tab **Desktop**.
2. Pada kolom URL, ketikkan IP Address milik Server: `192.168.2.1`, lalu klik tombol **Go**.
3. Apabila layar menampilkan halaman web Cisco Packet Tracer, maka arsitektur **Client-Server** Anda telah berhasil dikonfigurasi dengan sempurna.
