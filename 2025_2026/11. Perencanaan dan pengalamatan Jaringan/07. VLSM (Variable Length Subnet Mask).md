# ⚙️ Materi 7: Variable Length Subnet Mask (VLSM)

Pada materi Subnetting konvensional (Materi 5), kita membagi jaringan menjadi beberapa sub-jaringan dengan ukuran blok yang **sama besar** (disebut *Fixed Length Subnet Mask* atau FLSM). Namun, di dunia nyata, kebutuhan jumlah komputer di setiap ruangan sangat bervariasi. Untuk mengatasi masalah variasi kebutuhan ini tanpa membuang alokasi IP Address, kita menggunakan metode **VLSM (Variable Length Subnet Mask)**.

---

## 📌 1. Memahami Konsep VLSM

### Definisi

**VLSM (Variable Length Subnet Mask)** adalah teknik subnetting tingkat lanjut di mana sebuah jaringan besar dipecah menjadi beberapa subnet dengan **ukuran Subnet Mask yang berbeda-beda**, disesuaikan langsung dengan kebutuhan jumlah *host* riil di setiap segmen jaringan.

### Perbandingan FLSM vs VLSM

* **FLSM (Materi 5):** Jika satu subnet diatur untuk menampung 60 *host* (menggunakan prefix `/26`), maka seluruh subnet lainnya di dalam jaringan tersebut harus berukuran 60 *host*, meskipun ada ruangan yang hanya membutuhkan 2 buah IP. Hal ini menyebabkan pemborosan IP yang besar.
* **VLSM:** Memungkinkan kita memberikan prefix `/26` (62 host) untuk ruangan besar, prefix `/28` (14 host) untuk ruangan kecil, dan prefix `/30` (2 host) khusus untuk jalur penghubung antar-router.

---

## 🔍 2. Aturan Emas VLSM: Mengurutkan dari Terbesar ke Terkecil

Dalam menghitung dan membagi IP menggunakan metode VLSM, terdapat satu aturan mutlak yang wajib dipatuhi:

> ⚠️ **Aturan Utama:** Proses perhitungan alokasi Subnetting wajib dilakukan secara berurutan, dimulai dari **kebutuhan jumlah host yang paling besar** hingga menuju ke **kebutuhan host yang paling kecil**.

**Mengapa harus diurutkan?**
Jika perhitungan dimulai dari subnet kecil terlebih dahulu, maka alamat IP jaringan akan tumpang tindih (*overlapping*) dan merusak struktur blok subnet besar di bawahnya, sehingga router tidak dapat meneruskan paket data dengan benar.

## 🔢 3. Studi Kasus Perhitungan dan Pembagian IP VLSM

### Kasus:

Anda diberikan sebuah IP Network utama: **`192.168.1.0/24`** (Total kapasitas: 256 IP).
Anda diminta membagi jaringan tersebut untuk kebutuhan empat area di sekolah:

1. **Lab RPL:** 60 Host
2. **Lab TKJ:** 30 Host
3. **Ruang Guru:** 10 Host
4. **Link WAN (Jalur Antar Router):** 2 Host

### Langkah 1: Mengurutkan Kebutuhan Host

Urutan alokasi perhitungan: **Lab RPL (60)** ➔ **Lab TKJ (30)** ➔ **Ruang Guru (10)** ➔ **Link WAN (2)**.

### Langkah 2: Perhitungan Step-by-Step

#### 1. Lab RPL (Kebutuhan: 60 Host)

* Cari total host ($2^n$) yang paling mendekati dan mampu menampung 60 host. Didapatkan $2^6 = 64$ IP.
* Host efektif (usable): $64 - 2 = 62$ host (Mencukupi untuk kebutuhan 60).
* Sisa bit host = 6, maka Prefix CIDR = $32 - 6 =$ **`/26`** (Subnet Mask: `255.255.255.192`).
* **Alokasi Subnet 1:**
* *Network ID:* `192.168.1.0`
* *Host Range:* `192.168.1.1` s.d `192.168.1.62`
* *Broadcast ID:* `192.168.1.63`



#### 2. Lab TKJ (Kebutuhan: 30 Host)

* Alamat IP awal melanjutkan dari Broadcast sebelumnya, yaitu **`192.168.1.64`**.
* Cari total host ($2^n$) yang mendekati 30 host. Didapatkan $2^5 = 32$ IP.
* Host efektif (usable): $32 - 2 = 30$ host (Pas untuk kebutuhan 30).
* Sisa bit host = 5, maka Prefix CIDR = $32 - 5 =$ **`/27`** (Subnet Mask: `255.255.255.224`).
* **Alokasi Subnet 2:**
* *Network ID:* `192.168.1.64`
* *Host Range:* `192.168.1.65` s.d `192.168.1.94`
* *Broadcast ID:* `192.168.1.95`



#### 3. Ruang Guru (Kebutuhan: 10 Host)

* Alamat IP awal melanjutkan ke **`192.168.1.96`**.
* Cari total host ($2^n$) yang mendekati 10 host. Didapatkan $2^4 = 16$ IP.
* Host efektif (usable): $16 - 2 = 14$ host (Mencukupi untuk kebutuhan 10).
* Sisa bit host = 4, maka Prefix CIDR = $32 - 4 =$ **`/28`** (Subnet Mask: `255.255.255.240`).
* **Alokasi Subnet 3:**
* *Network ID:* `192.168.1.96`
* *Host Range:* `192.168.1.97` s.d `192.168.1.110`
* *Broadcast ID:* `192.168.1.111`



#### 4. Link WAN / Antar Router (Kebutuhan: 2 Host)

* Alamat IP awal melanjutkan ke **`192.168.1.112`**.
* Cari total host ($2^n$) yang mendekati 2 host. Didapatkan $2^2 = 4$ IP.
* Host efektif (usable): $4 - 2 = 2$ host (Pas untuk kebutuhan 2).
* Sisa bit host = 2, maka Prefix CIDR = $32 - 2 =$ **`/30`** (Subnet Mask: `255.255.255.252`).
* **Alokasi Subnet 4:**
* *Network ID:* `192.168.1.112`
* *Host Range:* `192.168.1.113` s.d `192.168.1.114`
* *Broadcast ID:* `192.168.1.115`

## 🏗️ 4. Mengonfigurasi VLSM di Cisco Packet Tracer

Mari kita implementasikan hasil perhitungan matematis di atas ke dalam simulator jaringan.

### Langkah-Langkah Pembuatan Topologi:

1. Siapkan perangkat keras berikut pada lembar kerja:
* **2 unit Router** (Tipe *2911*, beri nama Router-A dan Router-B).
* **2 unit Switch** (Tipe *2960*).
* **3 unit PC** (PC-RPL, PC-TKJ, dan PC-Guru).


2. Hubungkan perangkat sesuai dengan segmentasi area:
* **Router-A** terhubung ke Switch RPL, Switch RPL terhubung ke **PC-RPL**.
* **Router-B** terhubung ke Switch TKJ dan Switch Guru (melalui dua interface berbeda).
* Hubungkan **Router-A** langsung ke **Router-B** menggunakan kabel *Serial DCE* atau *Copper Cross-Over* (untuk jalur Link WAN).



### Langkah Pengisian IP Konfigurasi (Sesuai Hasil Hitung VLSM):

* **PC-RPL:** IP `192.168.1.2` | Subnet Mask `255.255.255.192` | Gateway `192.168.1.1`
* **PC-TKJ:** IP `192.168.1.66` | Subnet Mask `255.255.255.224` | Gateway `192.168.1.65`
* **PC-Guru:** IP `192.168.1.98` | Subnet Mask `255.255.255.240` | Gateway `192.168.1.97`

### Konfigurasi Interface Router:

1. **Router-A (Gateway Lab RPL & Link WAN):**
* Interface ke arah Lab RPL: IP `192.168.1.1` | Mask `255.255.255.192`
* Interface ke arah Router-B (Link WAN): IP `192.168.1.113` | Mask `255.255.255.252`


2. **Router-B (Gateway Lab TKJ, Ruang Guru & Link WAN):**
* Interface ke arah Lab TKJ: IP `192.168.1.65` | Mask `255.255.255.224`
* Interface ke arah Ruang Guru: IP `192.168.1.97` | Mask `255.255.255.240`
* Interface ke arah Router-A (Link WAN): IP `192.168.1.114` | Mask `255.255.255.252`

## 🧪 5. Menguji Konektivitas Seluruh Subnet

Karena arsitektur jaringan VLSM melibatkan segmen jaringan berskala luas dengan beberapa router, pastikan Anda mengaktifkan protokol routing dasar (seperti *Static Routing* atau *RIP*) pada kedua router agar mereka saling mengenal segmen luar terlebih dahulu.

### Langkah Uji Koneksi:

1. Klik pada **PC-RPL**, masuk ke menu **Command Prompt**.
2. Lakukan pengujian ping secara bertahap menuju wilayah lokal hingga menyeberang segmen:
* Ping ke Gateway lokal: `ping 192.168.1.1` (Memastikan kabel lokal aman).
* Ping melompati Router ke Lab TKJ: `ping 192.168.1.66`
* Ping melompati Router ke Ruang Guru: `ping 192.168.1.98`



### Analisis Hasil:

* Jika hasil menunjukkan status **`Reply from...`**, selamat! Desain teknik VLSM Anda telah berhasil berjalan secara optimal.
* Penggunaan variasi subnet mask (`.192`, `.224`, `.240`, dan `.252`) pada masing-masing sub-jaringan terbukti mampu menghemat ketersediaan sisa IP Address utama sekolah dari ancaman kehabisan alokasi ruang *host*.
