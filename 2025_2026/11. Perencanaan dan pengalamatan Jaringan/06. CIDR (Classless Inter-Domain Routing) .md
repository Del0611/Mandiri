# 🌐 Materi 6: CIDR (Classless Inter-Domain Routing) dan Notasi Prefix

Pada Materi 5, kita telah mempraktikkan dasar-dasar pembagian jaringan (*Subnetting*). Namun, penulisan Subnet Mask konvensional (seperti `255.255.255.0`) dinilai kurang efisien dan terlalu panjang. Untuk mempermudah efisiensi alokasi dan penulisan alamat jaringan, industri jaringan global beralih menggunakan metode **CIDR**.

---

## 📌 1. Memahami Konsep CIDR (Classless Inter-Domain Routing)

### Definisi

**CIDR (Classless Inter-Domain Routing)** adalah sebuah metode pengalamatan IP IP yang mengabaikan sistem pembagian kelas tradisional (Kelas A, B, dan C).

Sebelum CIDR diciptakan pada tahun 1993, alokasi IP sangat kaku. Jika sebuah organisasi membutuhkan 300 alamat IP, mereka terpaksa harus mengambil alokasi Kelas B (kapasitas 65.534 *host*) karena Kelas C (kapasitas 254 *host*) tidak mencukupi. Hal ini memicu pemborosan jutaan IP Address secara global.

Dengan adanya CIDR, pembagian dan alokasi IP Address dapat ditentukan secara fleksibel berdasarkan kebutuhan bit per bit yang spesifik, tanpa terikat oleh batas kaku kelas IP standar.

---

## ✍️ 2. Menuliskan IP dengan Format Prefix

CIDR memperkenalkan metode penulisan alamat jaringan yang ringkas menggunakan **Notasi Prefix**, yang ditandai dengan tanda garis miring (**/**) diikuti oleh sebuah angka di belakang IP Address.

Angka di belakang garis miring tersebut menunjukkan **jumlah bit bernilai 1** pada Subnet Mask, atau dengan kata lain, menunjukkan panjang dari *Network ID*.

### Contoh Perbandingan Penulisan:

* **Cara Lama (Classful):** IP `192.168.1.1` dengan Subnet Mask `255.255.255.0`
* **Cara Baru (CIDR/Prefix):** `192.168.1.1/24`

### Tabel Konversi Prefix CIDR ke Subnet Mask (Kelas C):

| Notasi Prefix | Representasi Bit Oktet Terakhir | Nilai Subnet Mask Desimal |
| --- | --- | --- |
| **/24** | `00000000` | `255.255.255.0` |
| **/25** | `10000000` | `255.255.255.128` |
| **/26** | `11000000` | `255.255.255.192` |
| **/27** | `11100000` | `255.255.255.224` |
| **/28** | `11110000` | `255.255.255.240` |
| **/29** | `11111000` | `255.255.255.248` |
| **/30** | `11111100` | `255.255.255.252` |

## 🔢 3. Menghitung Jumlah Host Berdasarkan CIDR

Untuk mengetahui kapasitas daya tampung perangkat komputer di sebuah Subnet, kita dapat menggunakan rumus matematika berbasis sisa bit *Host ID*. Karena total bit IPv4 adalah 32 bit, maka sisa bit untuk *host* adalah $32 - n$ (di mana $n$ adalah nilai prefix).

### Rumus Total Host:

$$2^{(32 - n)}$$

### Rumus Host yang Dapat Digunakan (Usable Host):

$$\text{Host Efektif} = 2^{(32 - n)} - 2$$

> ⚠️ *Catatan: Dikurangi 2 karena alamat pertama dialokasikan sebagai Network ID dan alamat terakhir sebagai Broadcast ID.*

### Contoh Contoh Perhitungan:

Sebuah gedung kantor memiliki alokasi IP **`192.168.10.0/27`**. Berapa jumlah komputer yang bisa terhubung?

1. **Hitung Sisa Bit Host:** $32 - 27 = 5$ bit.
2. **Hitung Total IP Jaringan:** $2^5 = 32$ alamat IP.
3. **Hitung Host Efektif:** $32 - 2 = 30$ Host.
4. **Kesimpulan:** Jaringan tersebut dapat menampung maksimal **30 perangkat komputer** (misal dari IP `192.168.10.1` sampai dengan `192.168.10.30`).

## 🏗️ 4. Mengimplementasikan CIDR pada Simulasi Jaringan

Perangkat lunak simulasi seperti **Cisco Packet Tracer** membutuhkan input desimal tradisional saat Anda melakukan konfigurasi IP, sehingga Anda harus mampu mengonversi nilai prefix CIDR ke bentuk desimal sebelum memasukkannya ke sistem.

### Kasus Simulasi:

Kita diminta membangun jaringan lokal untuk **Ruang Administrasi** yang membutuhkan 20 komputer menggunakan alokasi jaringan `192.168.20.0/27`.

### Langkah-Langkah di Cisco Packet Tracer:

1. **Menyusun Perangkat:**
* Letakkan **1 unit Switch** (Tipe *2960*).
* Letakkan **2 unit PC** (PC0 dan PC1) sebagai perwakilan pengujian.
* Hubungkan PC0 dan PC1 ke Switch menggunakan kabel **Copper Straight-Through**.


2. **Konversi Logika Prefix:**
* Berdasarkan tabel CIDR, prefix `/27` memiliki nilai desimal Subnet Mask: **`255.255.255.224`**.


3. **Mengisi IP Configuration di PC0:**
* Klik **PC0** > Masuk ke tab **Desktop** > Pilih **IP Configuration**.
* Isi **IPv4 Address**: `192.168.20.1`
* Isi **Subnet Mask**: `255.255.255.224` *(Ubah nilai default ujungnya dari .0 menjadi .224)*.


4. **Mengisi IP Configuration di PC1:**
* Klik **PC1** > Masuk ke tab **Desktop** > Pilih **IP Configuration**.
* Isi **IPv4 Address**: `192.168.20.2`
* Isi **Subnet Mask**: `255.255.255.224`


5. **Pengujian Jaringan (Validasi):**
* Buka **Command Prompt** pada PC0.
* Ketik perintah: `ping 192.168.20.2`
* Pastikan hasil keluaran menunjukkan status `Reply from 192.168.20.2: bytes=32...`.



---

> 📝 **Rangkuman Evaluasi:**
> CIDR dan notasi prefix mempermudah dokumentasi administrasi jaringan. Melalui implementasi di Cisco Packet Tracer, kita belajar bahwa penulisan ringkas `/27` di atas kertas harus diinput ke dalam perangkat simulator dalam format desimal komparatifnya, yakni `255.255.255.224`, agar sistem logika perutean data dapat mendeteksi batasan jaringan dengan tepat.
