
# 🌐 Materi Perencanaan Jaringan Terintegrasi Cisco Packet Tracer

# 📘 Capaian Pembelajaran (CP)

Mampu merancang topologi dan arsitektur jaringan sesuai kebutuhan serta menerapkan pengalamatan jaringan (subnetting, CIDR, dan VLSM) menggunakan simulasi jaringan.


# 🎯 Poin-Poin Capaian Pembelajaran (CP) – Jaringan Komputer Terintegrasi Cisco Packet Tracer

---

# 📌 1. Pemahaman Dasar Jaringan
- Memahami konsep dasar jaringan komputer
- Menjelaskan fungsi perangkat jaringan (router, switch, PC, server)
- Memahami tujuan perancangan jaringan

---

# 🌐 2. Topologi Jaringan
- Mengidentifikasi jenis topologi (star, bus, ring, mesh)
- Menentukan topologi sesuai kebutuhan kasus
- Membuat topologi jaringan menggunakan :contentReference[oaicite:0]{index=0}
- Menguji koneksi antar perangkat (ping test)

---

# 🏗️ 3. Arsitektur Jaringan
- Memahami konsep client-server
- Memahami konsep peer-to-peer
- Membangun jaringan client-server di Cisco Packet Tracer
- Membangun jaringan peer-to-peer sederhana
- Menguji komunikasi antar perangkat

---

# 🌍 4. IP Addressing (IPv4)
- Memahami konsep IP Address
- Mengidentifikasi subnet mask dan gateway
- Mengkonfigurasi IP pada perangkat di Cisco Packet Tracer
- Menguji koneksi jaringan menggunakan IP Address

---

# 🔢 5. Subnetting
- Memahami konsep subnetting
- Menghitung subnet dasar
- Membagi jaringan menjadi beberapa subnet
- Menerapkan hasil subnetting di Cisco Packet Tracer
- Menguji koneksi antar subnet

---

# 🌐 6. CIDR (Classless Inter-Domain Routing)
- Memahami konsep CIDR
- Menuliskan IP dengan format prefix (/24, /25, dll.)
- Menghitung jumlah host berdasarkan CIDR
- Mengimplementasikan CIDR pada simulasi jaringan

---

# ⚙️ 7. VLSM (Variable Length Subnet Mask)
- Memahami konsep VLSM
- Mengurutkan kebutuhan host dari terbesar ke terkecil
- Membagi IP sesuai kebutuhan tiap jaringan
- Mengkonfigurasi VLSM di Cisco Packet Tracer
- Menguji konektivitas seluruh subnet

---

# 🖥️ 8. Konfigurasi Perangkat Jaringan
- Menghubungkan perangkat (PC, switch, router)
- Mengatur IP pada setiap perangkat
- Menggunakan kabel jaringan yang sesuai
- Menguji koneksi jaringan (ping, simulation mode)

---

# 🧪 9. Pengujian dan Troubleshooting
- Melakukan pengecekan koneksi jaringan
- Mengidentifikasi kesalahan konfigurasi
- Memperbaiki kesalahan IP atau subnet
- Memastikan seluruh perangkat terhubung dengan benar

---

# 🧩 10. Desain Jaringan Lengkap (Project)
- Menganalisis kebutuhan jaringan sederhana
- Membuat desain topologi lengkap
- Menerapkan IP addressing, subnetting, CIDR, dan VLSM
- Mengimplementasikan seluruh desain di Cisco Packet Tracer
- Menguji dan mempresentasikan hasil jaringan

---

# 📌 Kesimpulan
Setelah seluruh poin tercapai, kemampuan yang diharapkan adalah:
- Mampu merancang jaringan komputer sederhana hingga kompleks
- Mampu mengimplementasikan jaringan menggunakan simulasi
- Mampu menganalisis dan memperbaiki kesalahan jaringan

> 🌐 “Kemampuan jaringan tidak hanya dipahami, tetapi harus bisa dipraktikkan secara langsung.”

---

# 🎯 Tujuan Pembelajaran (TP)

1. Memahami konsep topologi jaringan melalui simulasi.
2. Mengidentifikasi kebutuhan jaringan pada studi kasus.
3. Menentukan topologi jaringan menggunakan Cisco Packet Tracer.
4. Merancang arsitektur jaringan client-server dan peer-to-peer.
5. Mengkonfigurasi perangkat jaringan di Cisco Packet Tracer.
6. Memahami dan menerapkan IPv4 pada perangkat simulasi.
7. Melakukan perhitungan subnetting dan mengujinya di Cisco.
8. Menerapkan CIDR dalam perancangan jaringan.
9. Menerapkan VLSM untuk efisiensi alamat IP.
10. Membuat desain jaringan lengkap dan mengujinya secara simulasi.
