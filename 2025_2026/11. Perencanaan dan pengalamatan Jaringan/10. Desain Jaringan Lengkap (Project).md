# 🧩 Materi 10: Desain Jaringan Lengkap (Projek Akhir Mandiri)

Bagian ini merupakan puncak dari seluruh rangkaian materi jaringan komputer menggunakan Cisco Packet Tracer. Siswa/mahasiswa akan diuji kemampuannya dalam menganalisis kebutuhan dunia nyata, menghitung efisiensi IP, membangun topologi, hingga melakukan *troubleshooting* akhir.

Berikut adalah 5 pilihan projek terintegrasi yang dirancang dengan berbagai tingkat kesulitan dan studi kasus yang relevan dengan kebutuhan industri maupun institusi pendidikan:

## 💻 Projek 1: Infrastruktur Jaringan Laboratorium Komputer SMK (Tingkat: Dasar)

### 1. Studi Kasus & Analisis Kebutuhan

Sebuah sekolah menengah kejuruan ingin membangun jaringan untuk 2 laboratorium baru, yaitu **Lab Software Engineering (RPL)** dan **Lab Jaringan (TKJ)**. Masing-masing laboratorium membutuhkan 25 unit komputer client. Kedua laboratorium harus terisolasi satu sama lain demi kelancaran ujian, namun keduanya harus bisa mengakses satu Server Nilai Pusat.

* **Kebutuhan Perangkat:** 50 PC Client, 2 Switch, 1 Server, 1 Router.
* **Alokasi IP Utama:** `192.168.10.0/24`

### 2. Skema Pengalamatan Jaringan (Subnetting FLSM)

Metode yang digunakan adalah Subnetting dasar dengan membagi jaringan menjadi blok sama besar menggunakan prefix **/26** (Kapasitas 62 usable host per subnet).

* **Subnet 1 (Lab RPL):** Network `192.168.10.0/26` | Gateway: `192.168.10.1`
* **Subnet 2 (Lab TKJ):** Network `192.168.10.64/26` | Gateway: `192.168.10.65`
* **Subnet 3 (Server Pusat):** Network `192.168.10.128/26` | IP Server: `192.168.10.130`

### 3. Langkah Implementasi & Pengujian

1. Buat topologi Star pada masing-masing Lab menggunakan Switch 2960.
2. Hubungkan kedua Switch Lab dan satu Server ke interface Router 2911 (Port `Gig0/0`, `Gig0/1`, dan `Gig0/2`).
3. Gunakan kabel *Copper Straight-Through* untuk seluruh koneksi.
4. **Pengujian:** Buka Command Prompt pada salah satu PC di Lab RPL, lakukan `ping` ke PC Lab TKJ (Harus berhasil melewati router) dan lakukan `ping` ke Server Nilai.

## 🏢 Projek 2: Sistem Jaringan Kantor Perusahaan Start-Up Pengembang Aplikasi (Tingkat: Menengah)

### 1. Studi Kasus & Analisis Kebutuhan

Sebuah perusahaan *software house* berskala menengah memerlukan perancangan jaringan terstruktur untuk kantor operasionalnya. Kantor tersebut dibagi menjadi 3 divisi dengan kebutuhan *host* yang berbeda:

1. **Divisi Programmer & Developer:** 14 Host
2. **Divisi Quality Assurance (QA) & UI/UX:** 6 Host
3. **Divisi Administrasi & HRD:** 6 Host

* **Kebutuhan Perangkat:** 3 Switch (satu per divisi), 1 Router Utama, 1 Server lokal (Git Repositori).
* **Alokasi IP Utama:** `192.168.1.0/24`

### 2. Skema Pengalamatan Jaringan (CIDR & VLSM)

Karena jumlah kebutuhan *host* antar-divisi berbeda signifikan, gunakan metode **VLSM** (Urutan dari host terbesar ke terkecil):

* **Divisi Developer (14 Host):** Membutuhkan blok 16 IP ➔ Prefix **/28** (Netmask: `255.255.255.240`). Network ID: `192.168.1.0/28`.
* **Divisi QA (6 Host):** Membutuhkan blok 8 IP ➔ Prefix **/29** (Netmask: `255.255.255.248`). Network ID: `192.168.1.16/29`.
* **Divisi Admin (6 Host):** Membutuhkan blok 8 IP ➔ Prefix **/29** (Netmask: `255.255.255.248`). Network ID: `192.168.1.24/29`.

### 3. Langkah Implementasi & Pengujian

1. Sambungkan masing-masing Switch divisi ke port interface Router Utama.
2. Masukkan konfigurasi IP Address dan Subnet Mask desimal yang unik pada setiap PC dan port gateway Router sesuai hitungan VLSM.
3. **Pengujian:** Aktifkan **Simulation Mode**. Kirimkan paket data dari Divisi Developer menuju Server lokal. Pastikan visualisasi jalannya paket data tidak mengalami *Drop* di tingkat Switch maupun Router.

## 🏛️ Projek 3: Integrasi Jaringan Multi-Gedung Kampus Terintegrasi (Tingkat: Mahir)

### 1. Studi Kasus & Analisis Kebutuhan

Sebuah institusi pendidikan tinggi memiliki dua gedung utama yang terpisah jarak cukup jauh: **Gedung A (Rektorat)** dan **Gedung B (Akademik)**. Masing-masing gedung memiliki Router lokal tersendiri, dan kedua gedung tersebut dihubungkan melalui jalur kabel *Point-to-Point* (Link WAN).

* **Gedung A (Rektorat):** Membutuhkan 100 IP Address untuk staf.
* **Gedung B (Akademik):** Membutuhkan 50 IP Address untuk dosen dan mahasiswa.
* **Link WAN (Antar Router):** Membutuhkan 2 IP Address.
* **Alokasi IP Utama:** `172.16.0.0/16`

### 2. Skema Pengalamatan Jaringan (VLSM Lanjut)

* **Subnet Gedung A (100 Host):** Diperlukan blok 128 IP ➔ Prefix **/25** (Netmask: `255.255.255.128`). Network: `172.16.0.0/25`.
* **Subnet Gedung B (50 Host):** Diperlukan blok 64 IP ➔ Prefix **/26** (Netmask: `255.255.255.192`). Network: `172.16.0.128/26`.
* **Link WAN (2 Host):** Diperlukan blok 4 IP ➔ Prefix **/30** (Netmask: `255.255.255.252`). Network: `172.16.0.192/30`.

### 3. Langkah Implementasi & Pengujian

1. Gunakan kabel *Copper Cross-Over* untuk menghubungkan Router-GedungA ke Router-GedungB melalui interface GigabitEthernet.
2. Konfigurasikan **Static Routing** pada kedua router agar jaringan Gedung A dapat mengenali jaringan Gedung B, dan sebaliknya.
* *Sintaks di Router A:* `ip route 172.16.0.128 255.255.255.192 172.16.0.194`


3. **Pengujian:** Lakukan `tracert` (Traceroute) dari PC di Gedung A menuju PC di Gedung B. Amati lompatan (*hop*) paket data untuk memastikan paket benar-benar melewati jalur WAN `/30` yang telah dikonfigurasi.

## 🌐 Projek 4: Sistem Keamanan Server DMZ Sederhana (Tingkat: Mahir)

### 1. Studi Kasus & Analisis Kebutuhan

Sebuah instansi ingin membangun jaringan yang aman untuk sistem informasinya. Mereka memiliki dua segmen internal: **Ruang Keuangan (Sensitif)** dan **Ruang Tamu/Guest (Umum)**, serta satu segmen khusus bernama **DMZ (Demilitarized Zone)** yang berisi Web Server sekolah yang dapat diakses oleh semua pihak. Jaringan harus dirancang agar pengguna dari Ruang Tamu bisa mengakses Web Server, tetapi tidak bisa masuk ke jaringan Ruang Keuangan.

* **Kebutuhan Perangkat:** 1 Router, 3 Switch, 2 PC, 1 Server.
* **Alokasi IP Utama:** `192.168.100.0/24` dengan pembagian CIDR bebas berorientasi keamanan.

### 2. Skema Pengalamatan Jaringan

* **Subnet Keuangan:** `192.168.100.0/27` (Prefix /27, kapasitas 30 host)
* **Subnet Guest:** `192.168.100.32/27` (Prefix /27, kapasitas 30 host)
* **Subnet DMZ (Server):** `192.168.100.64/29` (Prefix /29, kapasitas 6 host)

### 3. Langkah Implementasi & Pengujian

1. Hubungkan ketiga subnet ke tiga interface fisik yang berbeda pada Router Utama.
2. Aktifkan layanan HTTP pada Server di segmen DMZ.
3. **Pengujian Hak Akses:**
* Buka Web Browser pada PC Guest, ketikkan IP Web Server DMZ (`192.168.100.66`). Akses harus berstatus **Berhasil**.
* Buka Command Prompt pada PC Guest, lakukan `ping` ke PC Ruang Keuangan (`192.168.100.2`). Untuk membatasi akses ini secara penuh di tingkat lanjut, siswa diminta menganalisis fungsi dasar *Access Control List (ACL)* pada router.

## 🏫 Projek 5: Capstone Project – Smart School Infrastructure (Tingkat: Komprehensif / Final)

### 1. Studi Kasus & Analisis Kebutuhan

Siswa diminta untuk mendesain arsitektur jaringan terintegrasi untuk satu kawasan sekolah modern seutuhnya. Jaringan ini mengintegrasikan seluruh elemen sekolah ke dalam satu topologi terpadu:

1. **Gedung Pembelajaran (Lab Komputer & Kelas):** 120 Host.
2. **Ruang Administrasi & Tata Usaha:** 14 Host.
3. **Sistem Utama (Server Ujian CBT & Portal Akademik):** 2 Server.

* **Alokasi IP Utama:** `10.10.0.0/16` (Menggunakan fondasi IP Kelas A dengan kustomisasi Subnetting VLSM tingkat tinggi).

### 2. Skema Pengalamatan Jaringan (Rancangan Siswa)

Siswa harus melakukan analisis mandiri untuk menentukan nilai prefix yang paling efisien, mengurutkan kebutuhan dari 120 host, 14 host, hingga segmen server, serta memetakan batasan IP Network, IP Valid, dan Broadcast ID secara tertulis dalam laporan sebelum melakukan praktik.

### 3. Langkah Implementasi & Prosedur Evaluasi Kelulusan Projek

1. **Tahap Desain:** Siswa menggambar skema arsitektur topologi logis dan fisik secara mandiri di lembar kanvas Cisco Packet Tracer.
2. **Tahap Konfigurasi:** Menerapkan seluruh parameter IP, subnet mask hasil konversi prefix, dan gerbang default (*gateway*) pada seluruh komponen tanpa kesalahan.
3. **Tahap Troubleshooting:** Siswa wajib mendeteksi mandiri jika terjadi kegagalan sistem (*packet loss*) melalui bantuan penelusuran kontrol *OSI Layer* pada *Simulation Mode*.

## 📋 Kriteria Penilaian dan Format Presentasi Hasil Projek

Setelah siswa menyelesaikan salah satu projek di atas, mereka diwajibkan untuk mempresentasikan hasilnya dengan kriteria penilaian sebagai berikut:

1. **Ketepatan Analisis & Perhitungan (Bobot 30%):** Keakuratan laporan matematis dalam pembagian IP Address menggunakan metode Subnetting/CIDR/VLSM (Tidak boleh ada IP yang tumpang tindih/*overlapping*).
2. **Kerapian & Validitas Topologi (Bobot 20%):** Ketepatan dalam pemilihan jenis kabel jaringan (Straight vs Cross) dan penempatan port interface pada Cisco Packet Tracer.
3. **Fungsionalitas Sistem (Bobot 30%):** Keberhasilan pengujian konektivitas akhir yang dibuktikan dengan hasil pengujian `ping` (0% loss) dan fungsionalitas layanan (*Web Browser/Server Services*).
4. **Kemampuan Presentasi & Troubleshooting (Bobot 20%):** Kelancaran dalam menjelaskan alur jalannya data dan ketangkasan dalam menemukan solusi apabila penguji dengan sengaja mengubah salah satu parameter IP untuk mensimulasikan gangguan (*trouble*).
