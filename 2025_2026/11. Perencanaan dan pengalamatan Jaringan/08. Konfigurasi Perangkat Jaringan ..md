# 🖥️ Materi 8: Konfigurasi Perangkat Jaringan pada Cisco Packet Tracer

Setelah menguasai aspek logika pengalamatan jaringan melalui metode Subnetting, CIDR, dan VLSM, tahap selanjutnya adalah mengimplementasikan rancangan tersebut ke dalam perangkat keras melalui simulasi. Konfigurasi yang tepat pada media fisik (kabel) dan logika (IP Address) menentukan keberhasilan transmisi data.

---

## 📌 1. Menggunakan Kabel Jaringan yang Sesuai (Physical Layer)

Sebelum menghubungkan perangkat, Anda harus menentukan jenis kabel yang tepat. Di dalam Cisco Packet Tracer, kesalahan pemilihan kabel akan mengakibatkan kegagalan total pada lapisan fisik (ditandai dengan indikator kabel berwarna merah).

Secara umum, terdapat dua jenis kabel tembaga (*copper*) yang paling sering digunakan:

### A. Kabel Copper Straight-Through

* **Fungsi:** Menghubungkan **dua perangkat yang berbeda jenis** di dalam jaringan.
* **Contoh Implementasi:**
* PC ke Switch
* Router ke Switch
* Switch ke Access Point



### B. Kabel Copper Cross-Over

* **Fungsi:** Menghubungkan **dua perangkat yang sama jenis** (setingkat) di dalam jaringan.
* **Contoh Implementasi:**
* PC ke PC
* Switch ke Switch
* Router ke Router
* PC ke Router (Kedua perangkat ini dianggap setingkat karena sama-sama memproses Layer 3/Network Layer pada beberapa aspek fungsional).



### 📊 Tabel Matriks Pemilihan Kabel

| Perangkat A | Perangkat B | Jenis Kabel yang Wajib Digunakan |
| --- | --- | --- |
| **PC / Laptop** | **Switch** | Copper Straight-Through |
| **Router** | **Switch** | Copper Straight-Through |
| **PC / Laptop** | **Router** | Copper Cross-Over |
| **PC / Laptop** | **PC / Laptop** | Copper Cross-Over |
| **Switch** | **Switch** | Copper Cross-Over |
| **Router** | **Router** | Copper Cross-Over / Serial Cable |

## 🏗️ 2. Menghubungkan Perangkat di Cisco Packet Tracer

Berikut adalah langkah-langkah standar untuk membangun konektivitas fisik antar-perangkat:

1. Buka area kerja Cisco Packet Tracer.
2. Ambil perangkat yang dibutuhkan dari menu *Devices* (misalnya: 1 Router 2911, 1 Switch 2960, dan 2 PC).
3. Klik ikon **Connections** (simbol petir).
4. Pilih jenis kabel yang sesuai berdasarkan aturan matriks di atas.
5. Klik pada perangkat pertama, pilih port yang tersedia (contoh: `FastEthernet0` pada PC atau `GigabitEthernet0/0` pada Router).
6. Tarik kabel ke perangkat kedua, klik lalu pilih port tujuan (contoh: `FastEthernet0/1` pada Switch).

> ⚠️ **Indikator Lampu Port:**
> * **Merah:** Port dalam kondisi mati (*shutdown*) atau ada kesalahan tipe kabel.
> * **Oranye:** Switch sedang melakukan proses pembelajaran MAC Address (*Spanning Tree Protocol*). Tunggu sekitar 30 detik.
> * **Hijau:** Jalur fisik telah aktif dan siap mentransmisikan data.

## ⚙️ 3. Mengatur IP Address pada Setiap Perangkat

Konfigurasi logika dilakukan dengan menanamkan IP Address, Subnet Mask, dan Default Gateway pada setiap *interface* perangkat.

### A. Konfigurasi pada End-Device (PC / Server)

1. Klik pada ikon PC.
2. Masuk ke tab **Desktop** > pilih **IP Configuration**.
3. Pilih opsi **Static**.
4. Masukkan nilai parameter sesuai dengan hasil rancangan jaringan Anda:
* *IPv4 Address*: Alamat IP unik untuk PC tersebut.
* *Subnet Mask*: Batasan subnet (contoh: `255.255.255.0`).
* *Default Gateway*: Alamat IP Router yang menjadi pintu keluar jaringan lokal.



### B. Konfigurasi pada Interface Router

Router tidak akan mengaktifkan port-nya secara otomatis. Anda harus menyalakan dan mengonfigurasinya melalui dua cara:

* **Cara 1: Melalui Tab GUI (Config)**
1. Klik **Router** > masuk ke tab **Config**.
2. Pada menu *Interface*, klik port yang ingin diatur (misal: `GigabitEthernet0/0`).
3. Masukkan *IP Address* dan *Subnet Mask*.
4. Centang kotak **On** pada opsi *Port Status*.


* **Cara 2: Melalui Command Line Interface (CLI)**
Buka tab **CLI** pada Router, kemudian ketikkan perintah terstruktur berikut:
```text
Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

```

## 🧪 4. Menguji Koneksi Jaringan

Setelah seluruh perangkat terhubung secara fisik dan dikonfigurasi secara logika, langkah terakhir adalah melakukan validasi menggunakan dua metode utama di Cisco Packet Tracer.

### A. Metode Realtime (Ping Test)

Metode ini digunakan untuk mengetahui status konektivitas secara cepat melalui protokol ICMP.

1. Klik **PC0** > masuk ke tab **Desktop** > buka **Command Prompt**.
2. Ketik perintah `ping [IP_Target]`, contoh:

    ```bash
    ping 192.168.1.2
    ```
3.  Analisis hasil: Jika muncul pesan `Reply from...`, maka konfigurasi berhasil. Jika muncul `Request Timed Out (RTO)`, periksa kembali keselarasan IP dan status *Gateway* Anda.

### B. Metode Simulation Mode (Visual Paket Data)
Metode ini sangat berguna untuk keperluan edukasi dan analisis mendalam, karena Anda dapat melihat jalannya paket data secara *frame demi frame* melewati setiap perangkat.



1.  Ubah mode kerja dari **Realtime** ke **Simulation** di pojok kanan bawah layar (atau tekan tombol `Shift + S`).
2.  Klik ikon **Add Simple PDU** (simbol amplop tertutup di menu sebelah kanan atau tekan tombol `P`).
3.  Klik pada **PC Pengirim** sebagai sumber data, lalu klik pada **PC Tujuan** sebagai target pengiriman.
4.  Pada panel *Simulation Controls* di sebelah kanan, klik tombol **Play/Capture Forward** (ikon tanda panah ke kanan).
5.  Amati pergerakan amplop b visual tersebut:
    *   Paket akan bergerak dari PC ke Switch, lalu diteruskan ke perangkat tujuan.
    *   Jika berhasil sampai dan kembali dengan selamat, amplop akan menampilkan tanda centang (**V**) berwarna hijau.
    *   Jika gagal di tengah jalan, amplop akan menampilkan tanda silang (**X**) berwarna merah, dan Anda dapat mengeklik amplop tersebut untuk melihat status kegagalan pada lapisan OSI Layer tertentu.

## 📌 Kesimpulan
Proses konfigurasi perangkat jaringan menggabungkan ketepatan pemilihan media kabel pada *Physical Layer* dan keakuratan pengisian alamat pada *Network Layer*. Pemanfaatan *Simulation Mode* secara optimal akan sangat membantu Anda dalam melakukan *troubleshooting* mandiri sebelum jaringan diimplementasikan pada perangkat riil di lapangan.
