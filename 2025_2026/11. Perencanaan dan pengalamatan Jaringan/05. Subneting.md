# 🔢 Materi 5: Subnetting – Konsep, Perhitungan, dan Implementasi

Setelah menguasai dasar pengalamatan IPv4 pada Materi 4, kita dihadapkan pada tantangan efisiensi. Secara bawaan, IP kelas C memiliki 254 ruang *host*. Namun, bagaimana jika sebuah sekolah hanya membutuhkan 50 *host* untuk Lab A dan 50 *host* untuk Lab B? Jika menggunakan konfigurasi standar, akan terjadi pemborosan IP. Solusinya adalah menggunakan **Subnetting**.

---

## 📌 1. Memahami Konsep Subnetting

### Definisi

**Subnetting** adalah proses memecah satu jaringan besar (*Network*) menjadi beberapa jaringan kecil yang disebut **Subnet**.

### Mengapa Subnetting Diperlukan?

1. **Mengurangi Lalu Lintas Jaringan (*Reduce Broadcast Storm*):** Data *broadcast* hanya akan tersebar di dalam satu subnet saja, tidak ke seluruh jaringan besar, sehingga menghemat bandwidth.
2. **Meningkatkan Keamanan (*Security*):** Memungkinkan kita memisahkan jaringan antar-divisi (misal: memisahkan subnet ruang Guru dengan subnet Lab Siswa).
3. **Efisiensi Alokasi IP:** Menghindari pemborosan alamat IP yang tidak terpakai.

---

## 🔢 2. Menghitung Subnet Dasar (Studi Kasus Kelas C)

Untuk melakukan subnetting, kita harus mengubah nilai **Subnet Mask** standar. Kita akan meminjam beberapa bit dari bagian *Host ID* untuk dijadikan *Network ID* baru. Nilai ini direpresentasikannya melalui format **Prefix (CIDR)**, contohnya `/26`.

### Contoh Kasus:

Diketahui sebuah IP Network: **`192.168.1.0/26`**

Mari kita hitung parameter dasarnya:

1. **Analisis Subnet Mask /26 dalam Biner:**
* Artinya ada 26 angka desimal 1, dan sisanya 0 hingga total 32 bit.
* `11111111.11111111.11111111.11000000`
* Konversi ke desimal: **`255.255.255.192`**


2. **Rumus Mencari Jumlah Subnet:**

$$2^x$$



*(di mana $x$ adalah banyaknya angka 1 biner pada oktet terakhir)*.
Pada oktet terakhir `/26`, biner-nya adalah `11000000` (ada 2 angka 1).
* Jumlah Subnet = $2^2 = 4$ Subnet.


3. **Rumus Mencari Jumlah Host per Subnet:**

$$2^y - 2$$



*(di mana $y$ adalah banyaknya angka 0 biner pada oktet terakhir)*.
Pada oktet terakhir, ada 6 angka 0.
* Jumlah Host = $2^6 - 2 = 64 - 2 = 62$ Host yang dapat digunakan per Subnet.
* *Catatan:* Dikurangi 2 karena alamat pertama digunakan untuk **Network ID** dan alamat terakhir untuk **Broadcast ID**.


4. **Blok Subnet (Magic Number):**

$$256 - \text{nilai desimal oktet terakhir Subnet Mask}$$


* Blok Subnet = $256 - 192 = 64$.
* Maka subnet berikutnya kelipatan 64, yaitu: **0, 64, 128, 192**.

## 🏗️ 4. Menerapkan Hasil Subnetting di Cisco Packet Tracer

> ⚠️ **Aturan Emas:** Perangkat yang berada pada subnet yang berbeda **tidak akan bisa terhubung secara langsung** tanpa bantuan perangkat Layer 3, yaitu **Router**.

Kita akan mensimulasikan penghubungan **Subnet 1 (Lab A)** dan **Subnet 2 (Lab B)** menggunakan satu buah Router.

### Langkah-Langkah Pembuatan Topologi:

1. Siapkan perangkat berikut di kanvas kerja:
* **1 unit Router** (Pilih tipe *2911*)
* **2 unit Switch** (Tipe *2960*, satu untuk Lab A, satu untuk Lab B)
* **2 unit PC** (PC0 di Lab A, PC1 di Lab B)


2. Hubungkan perangkat menggunakan kabel **Copper Straight-Through**:
* PC0 ke Switch A, Switch A ke Router port `GigabitEthernet0/0`.
* PC1 ke Switch B, Switch B ke Router port `GigabitEthernet0/1`.



### Langkah Konfigurasi IP Perangkat:

* **Konfigurasi PC0 (Lab A - Menggunakan Subnet 1):**
* IP Address: `192.168.1.2`
* Subnet Mask: `255.255.255.192` *(Jangan gunakan mask default 255.255.255.0)*
* Default Gateway: `192.168.1.1` *(IP interface router terdekat)*


* **Konfigurasi PC1 (Lab B - Menggunakan Subnet 2):**
* IP Address: `192.168.1.66`
* Subnet Mask: `255.255.255.192`
* Default Gateway: `192.168.1.65`



### Langkah Konfigurasi Router (via GUI/Config Tab agar lebih mudah):

1. Klik **Router**, masuk ke tab **Config**.
2. Di bawah menu *INTERFACE*, klik **GigabitEthernet0/0** (Jalur Lab A):
* Isi IP Address: `192.168.1.1`
* Isi Subnet Mask: `255.255.255.192`
* Centang opsi **Port Status: On** *(Lampu indikator kabel akan berubah hijau)*.


3. Klik **GigabitEthernet0/1** (Jalur Lab B):
* Isi IP Address: `192.168.1.65`
* Isi Subnet Mask: `255.255.255.192`
* Centang opsi **Port Status: On**.

## 🧪 5. Menguji Koneksi Antar Subnet

Sekarang kedua komputer berada di subnet yang terisolasi namun dijembatani oleh Router.

### Langkah Pengujian:

1. Klik **PC0** (Lab A), buka **Command Prompt**.
2. Lakukan tes ping menyeberang subnet ke PC1 (Lab B):
```bash
ping 192.168.1.66

```



### Analisis Hasil Pengujian:

* **Pengiriman Pertama (*Timeout*):** Jangan panik jika pengiriman pertama atau kedua muncul tulisan *Request Timed Out*. Hal ini normal karena router sedang melakukan proses pemetaan alamat fisik (*ARP Resolution*).
* **Pengiriman Berikutnya (*Reply*):** Jalur data berikutnya akan menghasilkan respon `Reply from 192.168.1.66...`. Hal ini menandakan paket data berhasil dikirimkan dari Subnet 1, naik ke Router, lalu diturunkan secara akurat ke Subnet 2.
