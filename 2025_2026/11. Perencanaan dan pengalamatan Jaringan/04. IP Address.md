# 🌐 Materi 4: Pengalamatan IP (IPv4) dan Konfigurasi Dasar

Setelah memahami arsitektur jaringan pada Materi 3, langkah krusial berikutnya adalah memberikan "identitas" kepada setiap perangkat agar mereka dapat saling mengenali dan bertukar informasi. Identitas logis ini disebut dengan **IP Address**.

---

## 📌 1. Memahami Konsep IP Address (IPv4)

### Definisi

**IP (Internet Protocol) Address** adalah sebuah alamat identitas numerik yang unik yang diberikan kepada setiap perangkat (PC, Laptop, Server, Router) yang terhubung ke dalam jaringan komputer.

Jika dianalogikan dalam kehidupan sehari-hari, IP Address berfungsi layaknya **alamat rumah**. Tanpa adanya alamat yang jelas, surat atau paket (data) yang dikirimkan tidak akan pernah sampai ke tujuan.

### Struktur IPv4

IPv4 terdiri dari **32-bit bilangan biner** yang dibagi menjadi **4 bagian (oktet)**, di mana setiap oktet dipisahkan oleh tanda titik (*dot*). Untuk mempermudah manusia dalam membaca, bilangan biner tersebut dikonversi menjadi bilangan desimal (*Dotted Decimal Notation*).

* **Contoh Alamat IP:** `192.168.1.10`

Setiap IP Address selalu terbagi menjadi dua bagian utama:

1. **Network ID:** Bagian alamat yang menunjukkan identitas jaringan (seperti nama jalan komplek rumah).
2. **Host ID:** Bagian alamat yang menunjukkan identitas perangkat spesifik di dalam jaringan tersebut (seperti nomor rumah).

### Klasifikasi IPv4 (Secara Umum)

Untuk mempermudah manajemen distribusi, IPv4 dibagi menjadi beberapa kelas berdasarkan rentang oktet pertamanya:

| Kelas | Rentang Oktet Pertama | Default Subnet Mask | Peruntukan |
| --- | --- | --- | --- |
| **Kelas A** | 1 – 126 | 255.0.0.0 | Jaringan berskala sangat besar (Sedikit Network, Banyak Host) |
| **Kelas B** | 128 – 191 | 255.255.0.0 | Jaringan berskala menengah (Medium) |
| **Kelas C** | 192 – 223 | 255.255.255.0 | Jaringan berskala kecil / lokal (Banyak Network, Sedikit Host) |

## 📌 2. Mengidentifikasi Subnet Mask dan Gateway

IP Address tidak dapat berdiri sendiri; ia selalu berpasangan dengan **Subnet Mask** dan memerlukan **Gateway** untuk berkomunikasi keluar dari jaringan lokal.

### A. Subnet Mask

**Subnet Mask** adalah kombinasi angka numerik yang berfungsi untuk membedakan mana bagian *Network ID* dan mana bagian *Host ID* dari sebuah IP Address.

* **Contoh:** Jika sebuah PC memiliki IP `192.168.1.10` dengan Subnet Mask `255.255.255.0`, maka angka `255` menandakan bahwa tiga oktet pertama (`192.168.1.`) adalah **Network ID**, sedangkan angka `0` menandakan bahwa oktet terakhir (`.10`) adalah **Host ID** milik PC tersebut.
* Perangkat hanya bisa saling berkomunikasi secara langsung jika memiliki *Network ID* yang sama persis.

### B. Gateway (Default Gateway)

**Gateway** adalah alamat IP dari antarmuka (*interface*) **Router** yang menghubungkan jaringan lokal (LAN) dengan jaringan luar (WAN/Internet).

* **Analogi:** Gateway bertindak sebagai **pintu keluar gedung**. Jika sebuah PC ingin mengirim data ke PC lain di dalam satu ruangan (satu Network ID), data tersebut tidak memerlukan Gateway. Namun, jika PC ingin mengakses server Google di internet (berbeda Network ID), data tersebut wajib dikirimkan terlebih dahulu ke alamat Gateway agar diteruskan oleh Router.

## 🏗️ 3. Mengonfigurasi IP pada Perangkat di Cisco Packet Tracer

Berikut adalah panduan langkah demi langkah untuk mengonfigurasi IP Address secara statis (manual) pada perangkat akhir (*end-device*) di dalam Cisco Packet Tracer:

1. Buka topologi jaringan yang telah Anda buat di Cisco Packet Tracer.
2. Klik **satu kali** pada ikon perangkat yang ingin dikonfigurasi (misalnya **PC0**).
3. Jendela baru akan muncul. Pilih tab **Desktop** di bagian atas.
4. Klik pada menu **IP Configuration**.
5. Pada opsi pemilihan metode, pastikan memilih **Static**.
6. Isi kolom parameter dengan panduan sebagai berikut (Contoh Kelas C):
* **IPv4 Address:** `192.168.1.2`
* **Subnet Mask:** `255.255.255.0` (Biasanya akan terisi otomatis saat Anda menekan tombol Tab).
* **Default Gateway:** `192.168.1.1` (Alamat IP Router penghubung).


7. Tutup jendela konfigurasi. Pengaturan akan tersimpan secara otomatis.

## 🧪 4. Menguji Koneksi Jaringan Menggunakan IP Address

Setelah seluruh perangkat diberikan konfigurasi IP yang selaras, validasi jaringan harus dilakukan untuk memastikan pengalamatan bekerja dengan benar.

### Langkah-Langkah Pengujian via Command Prompt:

1. Klik pada **PC0**, masuk ke tab **Desktop** > **Command Prompt**.
2. Ketik perintah `ipconfig` untuk memverifikasi apakah IP yang Anda masukkan sudah terpasang dengan benar pada kartu jaringan (*Network Interface Card*).
3. Lakukan pengujian koneksi ke perangkat tujuan (misal PC1 dengan IP `192.168.1.3`) menggunakan perintah **Ping**:
```bash
ping 192.168.1.3

```



### Hasil Analisis Konfigurasi:

* **Berhasil (Reply):** Menunjukkan bahwa *Subnet Mask* antar-perangkat sudah berada di jaringan yang sama dan konfigurasi IP tidak mengalami konflik (tidak ada IP ganda).
* **Gagal (Request Timed Out / Destination Host Unreachable):** Menunjukkan adanya kesalahan logika pengalamatan. Periksa kembali apakah oktet *Network ID* pada IP Address Anda sudah sama dengan perangkat tujuan, atau periksa kembali akurasi pengisian angka pada *Subnet Mask*.
