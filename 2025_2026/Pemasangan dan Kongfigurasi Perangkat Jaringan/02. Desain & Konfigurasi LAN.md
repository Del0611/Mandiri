# Modul Praktik Lengkap: Desain & Konfigurasi LAN

## 1. Persiapan Lab & Skenario

Dalam dunia nyata, teknisi tidak langsung memasang kabel. Kita mulai dengan skenario bisnis:

* **Divisi Admin:** Membutuhkan akses cepat ke server data internal.
* **Divisi Teknis:** Membutuhkan bandwidth besar untuk pengujian perangkat.

---

## 2. Perancangan Topologi (Penjelasan Teknis)

* **Pemilihan Kabel:** Kita menggunakan kabel **Copper Straight-Through** karena kita menghubungkan perangkat yang berbeda (*berbeda layer*), yaitu PC (Layer 7/Application) ke Switch (Layer 2) dan Switch ke Router (Layer 3).
* **Fungsi Router (Gateway):** Router di sini berfungsi sebagai "pintu keluar" (Gateway). Tanpa Router, PC di Divisi Admin tidak akan pernah bisa mengirim data ke PC di Divisi Teknis karena mereka berada di jaringan (Subnet) yang berbeda.

---

## 3. Implementasi Pengalamatan (Analisis VLSM)

**Mengapa kita menggunakan `/28` dan `/27`?**

* **Subnetting/VLSM** bertujuan untuk efisiensi. Jika kita menggunakan `/24` (254 host) untuk divisi yang hanya butuh 10 host, maka kita membuang banyak alamat IP yang berharga.
* **Subnet Mask 255.255.255.240 (/28):** Menyediakan 14 alamat IP yang bisa dipakai. Sangat pas untuk Divisi Admin (10 host).
* **Subnet Mask 255.255.255.224 (/27):** Menyediakan 30 alamat IP yang bisa dipakai. Sangat pas untuk Divisi Teknis (20 host).

---

## 4. Konfigurasi Perangkat (Penjelasan Perintah CLI)

### A. Konfigurasi Router (Gateway)

Berikut adalah penjelasan dari perintah yang kita gunakan:

```bash
Router> enable                          # Masuk ke Mode Privileged (akses penuh)
Router# configure terminal              # Masuk ke Mode Global Configuration
Router(config)# hostname R-UTAMA        # Mengubah nama identitas perangkat

R-UTAMA(config)# interface g0/0         # Masuk ke setingan fisik lubang port g0/0
R-UTAMA(config-if)# ip address ...      # Memberikan "KTP" atau alamat pada pintu tersebut
R-UTAMA(config-if)# no shutdown         # Menyalakan port (secara default port router mati/merah)

```

### B. Konfigurasi End Device (PC)

* **IP Address:** Alamat unik PC tersebut.
* **Subnet Mask:** Penentu batas jaringan.
* **Default Gateway:** Alamat IP Router yang terhubung ke Switch divisi tersebut. **Ingat:** Tanpa gateway, PC hanya bisa "mengobrol" dengan teman satu divisinya saja.

---

## 5. Uji Koneksi dan Verifikasi (Troubleshooting)

Saat melakukan `ping`, ada beberapa kemungkinan hasil:

1. **Reply from...:** Koneksi berhasil (Sempurna).
2. **Request Timed Out (RTO):** Kabel terhubung, tapi konfigurasi IP/Gateway salah, atau perangkat tujuan mati.
3. **Destination Host Unreachable:** Router tidak tahu jalan menuju jaringan tersebut (biasanya salah konfigurasi di interface router).

### Tips Menggunakan Simulation Mode:

Gunakan **ikon amplop (PDU)** lalu klik tombol **Play** di pojok kanan bawah. Perhatikan bagaimana paket data bergerak dari PC ke Switch, naik ke Router, lalu turun ke Switch tujuan. Jika paket "disilang" merah di salah satu titik, berarti di situlah letak kesalahannya.
